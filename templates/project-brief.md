# Project Brief

> Fill this out before every Claude Code session.
> Paste it at the start of your conversation.
> Vague brief → vague tool. Precise brief → useful tool.

---

## The Problem

*What's frustrating you right now? What are you doing manually that takes too long?*

[Write 2-3 sentences. Be specific. Name the friction.]

---

## The Input

*What data does this tool need? Where does it come from?*

Examples:
- A Google Sheet at [URL]
- A folder of CSV files at ~/Downloads/sales/
- An email inbox I'll forward things to
- Numbers I'll type in manually

[Describe your input.]

---

## The Output

*What should the finished result look like?*

Examples:
- A formatted email sent to me every Monday at 8am
- A CSV file I can open in Excel
- A webpage I can open in my browser
- A printed table in the terminal

[Describe your output.]

---

## The Trigger

*When should this run?*

Examples:
- Manually (I run it when I need it)
- Every Monday at 8am automatically
- Every time a new file appears in a folder
- When I type a specific command

[Describe the trigger.]

---

## The Constraints

*What must this NOT do? Any systems it can't touch? Any format it must use?*

Examples:
- Must not modify the original data files
- Must work without internet access
- Output must match this specific format: [...]
- Must not require me to install anything new

[List your constraints, or write "None" if there are none.]
